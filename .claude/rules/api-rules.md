# API 开发规则

当编写或修改 API 相关代码时，遵循以下规则：

- 所有 API 端点必须验证输入参数（类型 + 范围）
- 敏感操作必须检查用户权限
- 返回统一的响应格式：`{ success: boolean, data?: T, error?: string }`
- 错误响应使用正确的 HTTP 状态码（400/401/403/404/500）
- 数据库查询必须参数化，禁止字符串拼接 SQL
- API 响应不暴露内部实现细节（如数据库字段名、堆栈信息）
- 列表接口必须支持分页（page + pageSize 或 cursor）
